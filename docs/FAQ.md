# ❓ Frequently Asked Questions

## System Questions

### Q: What is this system?
**A**: A 6-skill integrated system for mastering research papers in 5-7 days through:
- Deep pedagogical learning (Paper Tutor)
- Gamified testing & implementation (Paper Mastery Game)
- Intelligent coordination (Orchestrator)

### Q: Is this for any paper?
**A**: Works best with 8-20 page research papers. Very long papers (50+ pages) take more time, short ones less.

### Q: How long does it take?
**A**: 10-12 focused hours over 5-7 days for expert mastery. Adjust pace as needed.

### Q: Can I go faster?
**A**: Yes! Tell system "I'm moving quickly" and it will accelerate challenges.

### Q: Can I go slower?
**A**: Yes! Tell system "I need more time" and it will provide more examples.

### Q: Do I need to read all files?
**A**: No. Read README.md and this FAQ. Reference others as needed.

## Learning Questions

### Q: What if I don't understand a concept?
**A**: 
1. Ask Paper Tutor: "Explain [concept] more intuitively"
2. Request background: "What's the background for [concept]?"
3. Get examples: "Give me a concrete example"
4. Ask for analogies: "Use an analogy to explain this"

### Q: What if I fail a quiz?
**A**: 
1. That's okay - quizzes reveal gaps!
2. Get background resources
3. Read more explanation
4. Retry the quiz
5. Celebrate when you pass!

### Q: What if my code doesn't pass tests?
**A**:
1. Read the error message carefully
2. Use hint system: "Give me a hint"
3. Ask for help: "I'm stuck on this error"
4. Try a different approach
5. Ask for pseudocode if needed

### Q: I'm stuck on implementation. What do I do?
**A**:
1. Read the test error (it tells you what's wrong)
2. Ask for a hint (costs 1 XP, helps you learn)
3. Ask for the approach (explains how to solve)
4. Full solution costs 10 XP (but less learning)
5. Ask questions anytime

## Progress Questions

### Q: How do I know if I understand?
**A**: You can:
- [ ] Explain without jargon?
- [ ] Derive equations?
- [ ] Implement code?
- [ ] Pass all tests?
- [ ] Teach someone else?

If YES to all → You understand!

### Q: What's a good XP progression?
**A**: Roughly:
- Day 1: 0-50 XP (theory, understanding)
- Day 2: 50-100 XP (quizzes)
- Day 3: 100-200 XP (implementation)
- Day 4: 200-300 XP (challenges)
- Day 5: 300-500+ XP (capstone)

Varies by person - don't worry about "perfect" progression.

### Q: When should I move to the next concept?
**A**: When you:
- ✅ Pass the quiz (70%+)
- ✅ Implement the component (tests pass)
- ✅ Feel confident

The Orchestrator will ask: "Ready for next concept?"

## Usage Questions

### Q: How do I load the skills?
**A**: In Claude:
```
I want to use the Paper Mastery System from
github.com/shivashankarv/journal-club

Load the skills and help me master a research paper.
[Upload PDF]
```

### Q: What commands do I use?
**A**: Paper Tutor:
- "Next section"
- "Explain [concept]"
- "Show me the math"

Game:
- "PLAY [N]" (quiz)
- "START SANDBOX" (code)
- "run_tests()" (check code)
- "HARD MODE" (challenges)

### Q: Can I customize the system?
**A**: Yes! Tell system:
- Learning style (Theorist/Practitioner/Balanced)
- Pace (fast/normal/slow)
- Difficulty (easy/normal/hard)
- Feedback style (concise/detailed)

### Q: What if I'm confused about which skill to use?
**A**: Don't worry - the Orchestrator decides automatically!
- Just ask your question
- System activates right skill
- You don't think about it

## Troubleshooting

### Q: "I don't know what to do next"
**A**: Ask the Orchestrator: "What should I do next?"

### Q: "I'm stuck and frustrated"
**A**:
1. Take a break (seriously!)
2. Go back to basics
3. Ask for simpler explanation
4. Use more examples
5. Message system: "I need help"

### Q: "The system is too hard"
**A**: 
1. Slow down: "I need more time"
2. Reduce difficulty: "Easy mode"
3. More examples: "Give me examples"
4. Back to theory: "Explain the fundamentals"

### Q: "The system is too easy"
**A**:

2. Harder challenges: "HARD MODE"
3. Skip ahead: "Give me capstone"
4. Research extensions: "Show me research variants"
