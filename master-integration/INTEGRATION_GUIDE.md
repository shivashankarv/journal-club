### User's Single Action
```

### System's Response (Orchestrated)

**Moment 1: Orchestrator Activates**
```
Orchestrator analyzes:
  • Paper title, abstract, structure (theory-heavy? practice-heavy?)
  • User's first message (what do they care about?)
  • Previous learning style (if this is not their first paper)
  → Detects profile: Theorist? Practitioner? Balanced?
  → Plans optimal path
```

**Moment 2: Paper Tutor Presents**
```
Paper Tutor provides:
  • Feynman Summary (intuitive overview)
  • Paper Skeleton (key sections)
  • First Section Deep Dive (with permission)
  
User reads, engages, says "next section"
```

**Moment 3: Orchestrator Monitors**
```
Orchestrator tracks:
  • How many sections has user engaged with?
  • Is understanding growing?
  • Is user ready to verify understanding?
  → Decision point: "Time for a quiz?"
```

**Moment 4: Paper Mastery Game Launches**
```
Game activates:
  • Quiz on just-learned concepts
  • Immediate feedback (pass/fail/score)
  • If passing: "Ready to implement?"
  • If failing: Offer background resources
  
User takes quiz, understands gaps
```

**Moment 5: Back to Tutor or Forward to Code**
```
Orchestrator logic:
  IF quiz_score > 80% AND user_is_ready:
    → Game: Implement in sandbox
  ELIF quiz_score < 60%:
    → Tutor: Explain prerequisites
  ELSE:
    → Ask user: "Want more theory or try coding?"
```

**Moment 6: Rinse & Repeat**
```
This cycle repeats:
  Tutor Section → Quiz → Implement → Level Up → Next Concept
  
Orchestrator ensures smooth transitions, no decision fatigue
```

**Moment 7: Capstone & Extension**
```
After 15+ concepts:
  Game: Build complete model (capstone)
  Tutor: Extend to peptides (your domain)
  
User achieves MASTER status
