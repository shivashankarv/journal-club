## Pro Tips for Maximum Learning

### Before You Start
1. **Set expectations**: You will work 5-7 days, ~3 hours/day
2. **Clear the calendar**: Dedicated focus time (no multitasking)
3. **Have environment ready**: Paper + notebook + IDE + PyTorch
4. **Set a goal**: "I want to teach this to my lab" (forces deep understanding)

### During Theory Phase (Tutor)
1. **Pause often**: Ask yourself "Can I explain this?"
2. **Sketch diagrams**: Visualize concepts by hand
3. **Try to derive**: Work through equations before reading the answer
4. **Ask follow-ups**: If confused, ask in detail (system is patient)

### During Implementation Phase (Game)
1. **Never skip quizzes**: They reveal your gaps
2. **Write pseudo-code first**: Before real code
3. **Read test error messages**: They teach you
4. **Optimize after correctness**: Get it working first
5. **Challenge yourself**: Try hard mode even when basic works

### After Completion
1. **Teach someone else**: Explain to a peer or colleague
2. **Apply to your domain**: Extend for peptides
3. **Build something novel**: Implement a variant
4. **Write about it**: Blog post or documentation

## Troubleshooting

### "I'm stuck on a concept"
→ **Paper Tutor**: "Explain [concept] with examples"
→ **Game**: Retake quiz, pull background resources

### "The implementation isn't passing tests"
→ **Game**: Read test error messages carefully
→ **Paper Tutor**: "Show me pseudocode for [component]"
→ **Game**: Use hint system if truly stuck

### "I'm bored/not motivated"
→ **Switch skills**: Tutor getting dry? Switch to Game challenges
→ **Set milestone**: "I'll finish [N] concepts today"
→ **Gamify harder**: Unlock bonus challenges, chase achievements

### "I don't understand why they chose this design"
→ **Paper Tutor**: "Why did they use X instead of Y?"
→ **Compare alternatives**: Implement both, see the difference
→ **Research context**: What did prior work do?

### "I'm running out of time"
→ **Prioritize**: Focus on core concepts (skip extensions first)
→ **Skip sandbox**: Understand theory deeply, implement later
→ **Read efficiently**: Skim sections you already know, deep-dive on novel parts
